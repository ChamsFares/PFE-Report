\section{Cisco NDFC}
\subsection{History, Evolution and Positioning}
Cisco NDFC is based on DCNM (Data Center Network Manager), a legacy Java application that had reached end-of-life in April 2026. NDFC operates on Nexus Dashboard Container Platform, providing a modern React-based UI along with the \textbf{Easy Fabric} flow in which a single intent declaration produces the underlay and overlay configuration deployment to all switches \cite{NDFC_Product_Documentation}.
\subsection{Architecture and Key Capabilities}
\begin{itemize}
    \item \textbf{Day 0 – Fabric Provisioning}: POAP (PowerOn Auto Provisioning) for zero‑touch onboarding; CDP/LLDP discovery; template‑based underlay/overlay generation.
    \item \textbf{Day 1 – Policy Configuration}: Guided VRF/VNI creation, pool‑based L2VNI/L3VNI assignment, consistent Route Targets, interface groups.
    \item \textbf{Day 2 – Operations and Compliance}: Continuous compliance checks (running config vs intent); topology visualisation; coordinated NX‑OS upgrades.
\end{itemize}
